SaleStatus

Git Source

An ISaleV2 can be in one of 4 possible states and a linear progression is expected from an "in flight" status to an immutable definitive outcome.

  • Pending: The sale is deployed onchain but cannot be interacted with yet.
  • Active: The sale can now be bought into and otherwise interacted with.
  • Success: The sale has ended AND reached its minimum raise target.
  • Fail: The sale has ended BUT NOT reached its minimum raise target. Once an ISaleV2 reaches Active it MUST NOT return Pending ever again. Once an ISaleV2 reaches Success or Fail it MUST NOT return any other status ever again.
enum SaleStatus {
    Pending,
    Active,
    Success,
    Fail
}
